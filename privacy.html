Privacy & Security

After Dark is designed with privacy first. Here's exactly how it works.

How After Dark accesses Canvas

After Dark does not store your Canvas credentials or API tokens. Instead, it routes all API requests through your existing Canvas browser session using a content script injected into open Canvas tabs. This means:

  • You must be logged into Canvas in a browser tab for After Dark to work.
  • After Dark has the same access level as your logged-in Canvas session — no more, no less.
  • When you log out of Canvas, After Dark loses access automatically.

Data storage

After Dark stores the following data locally in your browser using chrome.storage.local:

  • Your Canvas URL(s) and institution name from the setup wizard
  • Your theme preference
  • Your Vault entries (saved courses and notes)
  • Card Designer preferences and Demo Mode field selections

None of this data is transmitted to any external server. It lives only in your browser profile.

No external servers

After Dark does not have a backend. There are no analytics, no telemetry, no crash reporting, and no usage tracking. The extension communicates only between your browser and your Canvas instance.

Permissions

After Dark requests the following Chrome permissions:

  • activeTab — to detect which Canvas tab is open
  • tabs — to find Canvas tabs for routing requests
  • storage — to save your preferences locally
  • host permissions for your Canvas domain — to inject the content script that handles API requests
